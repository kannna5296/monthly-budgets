  - 内容: 策定済み予算取得ボタン押下で `GET /api/budgets?year=YYYY&month=M` を呼び、レスポンスを受けてフォーム状態を置き換える（既存行/補正列含む）。編集後に Save で POST できること。
  - 受け入れ基準: Load→編集→Save の一連がブラウザで行え、保存後は API のデータが更新される。
- T032 UX: 保存ボタンの無効化と即時バリデーション
  - 内容: 全体の検証状態に基づき Save ボタンを disabled にする。個別セルの即時エラー表示を追加。
  - 受け入れ基準: フロントでベースや補正に非数値がある場合 Save が無効になり、該当セルにエラー表示が出ること。

 - T035 UI: カテゴリ合計表示（実装）
  - 内容: 表の各行に「合計」列を追加し、当該行のベース予算と全補正列の和（base + Σadjustments）をリアルタイムで計算して表示する。合計は編集不可の表示専用とし、数値フォーマットはカンマ区切り（例: "12,345 円"）とする。補正が空の場合はベースのみを表示し、補正セルに非数値が入っている場合はその補正を無視して合計を計算する（該当補正セルには個別エラーを出す）。API の保存 payload に合計は含めない。
  - 受け入れ基準: 表示が即時に更新されること、合計欄が編集不可であること、保存/読み込みの payload に合計フィールドが含まれないこと。

- T036 UI: 使えるお金（usable）と残額表示（実装）
  - 内容: 画面上に「使えるお金 = 収入 - 貯金目標」を表示し、その値からカテゴリごとの合計（ベース + 補正の全行合計）を差し引いた「残額」をリアルタイムで表示する。残額が負の場合は視覚的に明確な警告（赤いテキストやアイコン）を表示する。入力や行・補正列の追加・編集で即時更新されること。
  - 受け入れ基準: 「使えるお金」が[収入 - 貯金目標]で正しく表示されること。全カテゴリの割当合計が正しく集計され、残額 = 使えるお金 - 割当合計 が即時に更新されること。残額が負の時、明確な警告表示が出ること。API の payload は変更しない（集計値はクライアント計算）。
